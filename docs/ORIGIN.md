# Origin

This repo was created as part of the GTM Systems, Traffic Integrity, Digital Intelligence, and Platform Engineering expansion wave for the Kinetic Gain portfolio.

## Intent

The goal was to show that headless WordPress architecture belongs in the same conversation as speed, SEO, and editorial trust. Rather than treating a decoupled CMS as a vague technology choice, the repo turns it into a buildable starter:

- WordPress stays the editorial source of truth
- Vue owns frontend delivery and metadata control
- preview flow and GraphQL contracts are treated as first-class concerns

## Positioning

- role level: `Principal`

- focus: headless CMS, infrastructure, DevOps, core web vitals
